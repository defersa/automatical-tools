Меню

В момент заполнения полей таблицы Top / Side, при "переходе" из дерева переменных Variables в поля, переменная может принять одно из трех состояний:

Переменная (либо значение переменной )
Фильтр-переменная
Весовая переменная

Состояние переносимой переменной зависит от включенного режима заполнения поля.

Режим заполнения Вектор. При заполнении полей в данном режиме, переменные из дерева переменных переходят/добавляются в поля Top и Side не меняя своего состояния, то есть остаются переменными, (либо значение переменной ).
Однако, в режиме Вектор возможно применить ряд функций из списка Function, при этом переменная изменит свое состояние на функциональню переменную →
Режим заполнения Фильтр. При данном режиме при добавлении переменной из дерева переменных в поля Top / Side добавляемая переменная изменяет свое состояние на фильтр переменную → .
Фильтр переменная накладывает соответствующий фильтр на все элементы поля, если при переходе было активно все поле, либо на конкретные переменные (одну или несколько), если при переходе были выделены определенные переменные.

Также из дерева переменных одновременно можно добавлять несколько фильтр-переменных. В этом случаи фильтры объединяются через логическое И. Для того чтобы наложить на переменные (одну или несколько) несколько фильтр-переменных, объединенных через логическое ИЛИ, необходимо прописать перечисление фильтров через ИЛИ (|) в спецификации поля.

Открыть окно спецификации поля нужно кликнуть по соответствующей полю кнопке
Режим заполнения Вес. При данном режиме переменная из дерева переменных при добавлении в поля Top / Side преобразуется в весовую переменную → .
Аналогично фильтр-переменной, весовая переменная может накладывать вес, в соответствии весовой переменной, на все элементы поля или некоторые из них, в зависимости что было активным во время добавления весовой переменной.

А также, при выборе нескольких переменных в дереве переменных добавление нескольких весовых переменных, объединенных через логическое И, ИЛИ. По умолчанию, веса объединяются через И, для формировани яобъединия по ИЛИ требуется править код спецификации поля.

Режим заполнения Иерархия. Данный режим предназначен для группы переменных у которых значения переменных (ответы) являются одинаковыми по лейауту.
При добавлении таких групп переменных в поле, в которое осуществляется переход, добавляются переменные со ссылкой на значения переменных , которые автоматически добавляются в противоположное поле в единственном экземпляре. Ссылка на переменные или на значения переменных обозначается как ?.

Если в дереве переменных элементы с одинаковыми значениями переменных собраны в одной папке, при заполнении полей достаточно выбрать/выделить их общую папку.
